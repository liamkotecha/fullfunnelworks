# Prompt 09 — GA4 Integration: Portal Events & Measurement Protocol

## CONTEXT

You are adding Google Analytics 4 tracking to the Full Funnel Works portal. This covers two distinct things:

1. **Client-side events** — tracked from the portal using `gtag` as clients and consultants use the app
2. **Server-side events** — sent via GA4 Measurement Protocol from API routes when pipeline stage changes occur, closing the loop between the WordPress lead form and client conversion

This prompt depends on Prompts 06 and 07 being complete (Prospect model and pipeline must exist).

## STEP 2 — GA4 ANALYTICS LIBRARY

Create `src/lib/analytics.ts`:

```typescript
/**
 * analytics.ts — GA4 tracking utilities.
 *
 * TWO modes:
 *
 * 1. Client-side: trackEvent() — calls window.gtag() if available
 *    Used in portal pages and admin pages
 *    Never throws — silent fail if gtag not loaded
 *
 * 2. Server-side: trackServerEvent() — calls GA4 Measurement Protocol API
 *    Used in API routes after significant business events
 *    Never throws — silent fail on network error or missing config
 *    Returns void (fire-and-forget — do not await in critical paths)
 */

// ── Client-side ──────────────────────────────────────────────

/**
 * trackEvent — fire a GA4 event from the browser.
 * Safe to call even if GA4 is not configured — checks for gtag existence first.
 */
export function trackEvent(
  eventName: string,
  params?: Record<string, string | number | boolean>
): void {
  try {
    if (typeof window === "undefined") return
    if (typeof (window as any).gtag !== "function") return
    ;(window as any).gtag("event", eventName, params ?? {})
  } catch {
    // silent
  }
}

// ── Server-side ──────────────────────────────────────────────

export interface ServerEventPayload {
  measurementId: string     // G-XXXXXXXXXX
  apiSecret:     string
  clientId:      string     // GA4 client_id (from _ga cookie)
  sessionId?:    string     // GA4 session_id (optional, improves attribution)
  eventName:     string
  params?:       Record<string, string | number | boolean>
}

/**
 * trackServerEvent — send a GA4 event via Measurement Protocol.
 * Non-blocking — call without await in API routes.
 *
 * Endpoint: https://www.google-analytics.com/mp/collect
 * Docs: https://developers.google.com/analytics/devguides/collection/protocol/ga4
 */
export async function trackServerEvent(payload: ServerEventPayload): Promise<void> {
  try {
    const url = `https://www.google-analytics.com/mp/collect?measurement_id=${payload.measurementId}&api_secret=${payload.apiSecret}`
    await fetch(url, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        client_id:  payload.clientId,
        events: [{
          name:   payload.eventName,
          params: {
            session_id:   payload.sessionId,
            engagement_time_msec: "1",
            ...(payload.params ?? {}),
          },
        }],
      }),
    })
  } catch {
    // silent
  }
}

/**
 * getGA4Settings — fetches GA4 config from Settings model.
 * Cached module-level for 5 minutes to avoid DB hit on every request.
 * Returns null if GA4 not configured or disabled.
 */
let settingsCache: { data: GA4Config | null; fetchedAt: number } | null = null

export interface GA4Config {
  measurementId: string
  apiSecret:     string
  trackedEvents: Record<string, boolean>
}

export async function getGA4Settings(): Promise<GA4Config | null>
```

Implement `getGA4Settings` with the 5-minute cache. It should:
1. Check cache — return cached value if < 5 minutes old
2. Otherwise: `Settings.findOne()`, extract GA4 fields
3. Return null if `!ga4Enabled` or if `measurementId` or `apiSecret` are missing
4. Cache and return the config

## STEP 4 — SERVER-SIDE EVENTS: PIPELINE STAGE CHANGES

In `src/app/api/prospects/[id]/route.ts` (PATCH handler), after a successful stage change, fire a server-side event:

```typescript
// After saving prospect with new stage:
void (async () => {
  const config = await getGA4Settings()
  if (!config) return

  const eventMap: Partial<Record<ProspectStage, { eventName: string; configKey: string }>> = {
    sql:         { eventName: "lead_qualified",   configKey: "leadQualified" },
    proposal:    { eventName: "proposal_sent",    configKey: "proposalSent" },
    won:         { eventName: "deal_won",         configKey: "dealWon" },
    lost:        { eventName: "deal_lost",        configKey: "dealLost" },
  }

  const mapping = eventMap[newStage]
  if (!mapping) return
  if (!config.trackedEvents[mapping.configKey]) return
  if (!prospect.gaClientId) return   // can't track without browser client_id

  await trackServerEvent({
    measurementId: config.measurementId,
    apiSecret:     config.apiSecret,
    clientId:      prospect.gaClientId,
    eventName:     mapping.eventName,
    params: {
      business_name: prospect.businessName,
      lead_score:    prospect.leadScore,
      deal_value:    prospect.dealValue ? prospect.dealValue / 100 : 0,
      currency:      "GBP",
    },
  })
})()
```

Note the `void (async () => { ... })()` pattern — this fires the event without blocking the API response.

## STEP 5 — CLIENT-SIDE EVENTS: PORTAL ACTIONS

Create a portal analytics hook `src/hooks/usePortalAnalytics.ts`:

```typescript
"use client"

import { trackEvent } from "@/lib/analytics"

/**
 * usePortalAnalytics — provides event tracking functions for portal pages.
 * All functions are safe to call — silent fail if GA4 not loaded.
 */
export function usePortalAnalytics() {
  return {
    trackAssessmentStarted: (section: string) =>
      trackEvent("assessment_started", { section }),

    trackSectionCompleted: (section: string, subSection: string, score: number) =>
      trackEvent("section_completed", { section, sub_section: subSection, completion_score: score }),

    trackModuleCompleted: (moduleId: string) =>
      trackEvent("module_completed", { module_id: moduleId }),

    trackReportDownloaded: () =>
      trackEvent("report_downloaded"),

    trackPortalPageView: (pageName: string) =>
      trackEvent("portal_page_view", { page_name: pageName }),
  }
}
```

```
GA4 Funnel Tracking Setup

To track leads from form submission through to client conversion, add two
hidden fields to your Elementor form. These capture the visitor's GA4
```
