TECH STACK
- Framework:    Next.js 14 (App Router)
- Language:     TypeScript
- Database:     MongoDB via Mongoose
- Auth:         Custom (password, OTP, WebAuthn/passkey)
- Styling:      Tailwind CSS
- Email:        SendGrid
- Payments:     Stripe
- Analytics:    GA4
- Hosting:      (Vercel-compatible)

--------------------------------------------------------------------------------
USER ROLES
--------------------------------------------------------------------------------
  1. admin       — Full platform access. Manages consultants, plans, billing,
                   CRM, questions, settings, and all client/project data.
  2. consultant  — Scoped to assigned clients. Manages their own projects,
                   invoices, and CRM pipeline.
  3. client      — Portal access only. Completing the growth framework,
                   viewing their roadmap, KPIs, financial model, invoices.

--------------------------------------------------------------------------------
ROUTING & AUTH MIDDLEWARE
--------------------------------------------------------------------------------
  / (root)         → Redirects by role:
                       admin       → /admin/dashboard
                       consultant  → /consultant/dashboard
                       client      → /portal/overview
                       (unauth)    → /login

  /admin/*         → Blocked if not admin
  /consultant/*    → Blocked if not consultant or admin
  /portal/*        → Client-only (consultants/admins can view read-only)

  Public routes:   /login, /register, /pricing, /verify

ROLE 2 — CONSULTANT (/consultant/*)
================================================================================

Consultants see a scoped version of the admin interface — only their own
clients, projects, prospects, and invoices. They cannot access plans, all-
consultant workload, settings, or another consultant's data.

────────────────────────────────────────────────────────────────────────────────
  DASHBOARD  /consultant/dashboard
────────────────────────────────────────────────────────────────────────────────
  What it shows:
  - Blocked Projects widget: projects with open blocks / stalled status
  - Recent Clients list: their most recently active clients with quick links
  - Overdue Invoices widget: invoices past due date

  Actions available:
  - Click project → project detail
  - Click client → client detail
  - Click invoice → invoice detail

────────────────────────────────────────────────────────────────────────────────
  CLIENTS  /consultant/clients
────────────────────────────────────────────────────────────────────────────────
  Same as admin client list but filtered to this consultant's assigned clients.

  Actions available:
  - Click client → /consultant/clients/[id]

────────────────────────────────────────────────────────────────────────────────
  CLIENT DETAIL  /consultant/clients/[id]
────────────────────────────────────────────────────────────────────────────────
  Identical structure to admin client detail:
  - Overview: company details, projects, active modules, assessment team
  - Sub-pages: /activity, /responses, /synthesis, /onboarding, /kpis,
    /roadmap, /financial-model, /hiring-plan, /invoices

  The consultant can view and edit responses, manage KPIs, update
  the roadmap, and generate/manage invoices for their own clients.

  Actions available:
  - Same as admin client detail but scoped (cannot reassign client to
    another consultant, cannot change plan/module settings)

────────────────────────────────────────────────────────────────────────────────
  PROJECTS  /consultant/projects/[id]
────────────────────────────────────────────────────────────────────────────────
  Same structure as admin project detail. Consultant can:
  - View and update framework responses
  - Navigate revenue-execution and execution-planning sub-sections
  - Manage milestones
  - Flag/unflag blocks

  Sub-sections (same as admin):
  - /revenue-execution + 8 sub-pages
  - /execution-planning + 7 sub-pages

────────────────────────────────────────────────────────────────────────────────
  CRM PIPELINE  /consultant/crm/pipeline
────────────────────────────────────────────────────────────────────────────────
  Consultant's own prospect pipeline — same kanban layout as admin:
  MQL → SQL → Discovery → Proposal → Negotiating → Won / Lost

  Actions available:
  - Drag cards between stages
  - Click card → prospect detail (their own prospects only)
  - (Cannot see other consultants' prospects)

────────────────────────────────────────────────────────────────────────────────
  INVOICES  /consultant/invoices
────────────────────────────────────────────────────────────────────────────────
  What it shows:
  - KPI cards at top: Total Outstanding, Total Paid, Overdue count
  - Sortable table of all invoices for their clients
  - Columns: client name, invoice number, amount, status, due date
  - Row actions: view, edit, mark paid, send reminder

  Actions available:
  - Create invoice (new invoice for one of their clients)
  - View invoice detail → /consultant/invoices/[id]
  - Mark invoice as paid
  - Send payment reminder email
  - Edit invoice amount/due date/items

  /consultant/invoices/[id]
  - Full invoice detail with line items
  - Status management (draft → sent → paid / overdue)

ROLE 3 — CLIENT PORTAL (/portal/*)
================================================================================

The portal is the client-facing growth framework completion interface.
Clients work through sections of the framework, see their progress,
and access the financial modeller and invoices.

────────────────────────────────────────────────────────────────────────────────
  OVERVIEW  /portal/overview
────────────────────────────────────────────────────────────────────────────────
  What it shows:
  - Time-of-day greeting: "Good morning/afternoon/evening, [name]"
  - Progress "speedometer" gauge: overall completion % across all sections
  - Resume card: "Continue where you left off" — shows the most recent
    incomplete section with a direct link
  - Section grid: all 11 framework sections as cards with:
      - Section name and icon
      - RAG status badge:
          Not started (grey) | In progress (amber) |
          On track (blue) | Complete (green)
      - Click → enters that section
  - In-progress tracker: list of sections currently in progress

  Actions available:
  - Click section card → section entry page
  - Click resume card → last active sub-page
  - (Read-only dashboard — no form inputs)

────────────────────────────────────────────────────────────────────────────────
  INTAKE  /portal/intake
────────────────────────────────────────────────────────────────────────────────
  Purpose: Initial onboarding intake form capturing company background.
  This is typically the starting point for new clients.

  What it shows:
  - Section pill navigation (animated slide between sections)
  - One section visible at a time, smooth transition animations
  - Auto-save: responses saved after each section completion
  - Progress bar showing intake completion
  - Submit-all action at final section

  Intake sections (in order):
    overview, assessment, people, product, process, roadmap, kpis, gtm,
    revenue_execution, execution_planning

  Question types used: text inputs, selects, sliders, checkboxes

  Actions available:
  - Next section (auto-saves current section)
  - Previous section
  - Submit (final section)
  - (Cannot skip sections — sequential)

────────────────────────────────────────────────────────────────────────────────
  ASSESSMENT  /portal/assessment/*
────────────────────────────────────────────────────────────────────────────────
  Sub-pages:

  /checklist
    - Assessment preparation checklist
    - Checkbox list of items to complete before deep-dive
    - Progress shown as completed / total

  /swot
    - SWOT Analysis: 4 quadrants (Strengths, Weaknesses, Opportunities,
      Threats)
    - Each entry has a text field + weight slider (1–5 importance)
    - Multiple items per quadrant
    - Save button per quadrant

  /most
    - MOST Analysis: Mission, Objectives, Strategy, Tactics
    - Structured form per MOST element
    - Descriptive text inputs + supporting detail fields

  /gap
    - Gap Analysis form
    - Current state vs desired state per area
    - Gap description and severity rating

  /leadership
    - Leadership assessment questions
    - Slider responses (1–5 or 1–10 scale)
    - Text commentary fields

  Actions available across all assessment sub-pages:
  - Save responses
  - Navigate between sub-pages (sequential progress tracking)

────────────────────────────────────────────────────────────────────────────────
  FINANCIAL MODELLER  /portal/modeller
────────────────────────────────────────────────────────────────────────────────
  3 tabs:

  Tab 1 — Base Model
    - Input current P&L: revenue, cost of sales, headcount costs, overhead
    - Growth assumptions: revenue growth %, margin targets
    - Auto-calculates: gross margin, EBITDA, runway
    - Save base model

  Tab 2 — Scenarios
    - Create named scenarios with 4 types:
        hire      — Add a hire: role, salary, start date, revenue impact
        price     — Price change: % increase/decrease, volume impact
        revenue   — New revenue stream: amount, start date, confidence %
        client    — New client win: contract value, start date, probability
    - Each scenario shows incremental P&L impact
    - Multiple scenarios can be built and named

  Tab 3 — Compare
    - Side-by-side comparison of base model vs scenarios
    - Charts: revenue over time, EBITDA over time
    - Scenario toggle (show/hide individual scenarios)
    - Summary delta table

  Actions available:
  - Save base model
  - Add/edit/delete scenarios
  - Toggle scenarios in compare view

  HIRING PLAN MODELLER  /portal/modeller/hiring

────────────────────────────────────────────────────────────────────────────────
  LEAD SCORING ALGORITHM  (0–100 points)
────────────────────────────────────────────────────────────────────────────────
  Company size (0–25 pts):
    1–10 employees     →  5 pts
    11–50              → 10 pts
    51–200             → 18 pts
    201–500            → 22 pts
    500+               → 25 pts

  Annual revenue (0–25 pts):
    Pre-revenue        →  0 pts
    Under £500k        →  8 pts
    £500k–£2m          → 15 pts
    £2m–£10m           → 22 pts
    £10m+              → 25 pts

  Primary challenge (0–25 pts):
    Any value submitted → 15 pts base
    High-intent keywords (scale, revenue, growth, sales, gtm, pipeline,
    team, hiring, strategy, market) → +10 pts additional

  Completeness (0–25 pts):
    +5 each for: phone provided, website provided, how they heard provided,
    message over 20 characters, primary challenge provided

────────────────────────────────────────────────────────────────────────────────
  AUTO-ASSIGNMENT ENGINE
────────────────────────────────────────────────────────────────────────────────
  Triggered when a new prospect/lead is created (webhook or manual).
  Algorithm (weighted round-robin with checks):
    1. Get all active consultants with available capacity
       (active clients < maxActiveClients from plan)
    2. Filter by specialism match: does consultant's specialism keywords
       overlap with prospect's primaryChallenge?
    3. Filter out consultants on holiday/unavailable
       (expired holidays cleaned up automatically)
    4. Sort by current active client count (ascending — least loaded first)
    5. Assign to first eligible consultant
    6. Log assignment to AssignmentLog model with reason/score
  
  If no eligible consultant found → prospect left unassigned, admin notified.

────────────────────────────────────────────────────────────────────────────────
  STALENESS ENGINE (Project Health)
────────────────────────────────────────────────────────────────────────────────
  Tracks time since last meaningful activity on a project.
  States:
    active      — Activity within expected window
    nudge       — Getting stale but manageable
    stalled     — No recent activity, needs attention
    at_risk     — Significantly stalled, consultant intervention needed
    terminated  — Project effectively inactive

  Used for:
  - Blocked Projects widget on consultant dashboard
  - "Needs Attention" list on admin dashboard
  - Project health badges on project lists

────────────────────────────────────────────────────────────────────────────────
  DIVERGENCE ENGINE (Team Response Analysis)
────────────────────────────────────────────────────────────────────────────────
  Measures disagreement between team members who complete the same framework
  questions (0 = unanimous, 1 = maximum disagreement).

  Calculation by question type:
    Slider questions:   Variance-based calculation across respondents' values
    Checkbox questions: Percentage of respondents who selected / didn't select
    Text questions:     0.5 if all non-empty but all different responses;
                        0 if all the same

  Shown in:
  - Admin synthesis page (/admin/clients/[id]/synthesis)
  - Consultant synthesis page (/consultant/clients/[id]/synthesis)
  - Responses grouped by sub-section with divergence score per question
  - High divergence flagged visually (threshold highlighting)

────────────────────────────────────────────────────────────────────────────────
  WEBHOOK LEAD INTAKE
────────────────────────────────────────────────────────────────────────────────
  Endpoint: POST /api/webhooks/lead
  Source: Elementor form on external website (or any webhook-compatible form)

  Flow:
    1. Receive POST with form data + webhook secret in header
    2. Validate secret against stored Settings.webhookSecret
    3. Parse 10 standard fields + 2 hidden fields:
       Standard: business_name, contact_name, email, phone, website,
                 company_size, revenue_range, challenge, hear_about, message
       Hidden:   ga_client_id, referrer_url
    4. Run lead scoring algorithm → score 0–100
    5. If auto-assign enabled → run assignment engine
    6. Create Prospect record in database
    7. Send notification email to admin (and assigned consultant if applicable)
    8. Return 200 OK
